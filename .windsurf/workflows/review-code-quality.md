# Code Quality Review

1. Check for unused imports in both frontend and backend
2. Verify consistent naming conventions (camelCase JS, snake_case Python)
3. Review type annotations in Python (type hints) and TypeScript (interfaces)
4. Check for code duplication - extract shared logic into utilities
5. Review file/folder structure - separation of concerns
6. Verify all API response shapes are consistent and documented
7. Check for TODO/FIXME/HACK comments that need resolution
